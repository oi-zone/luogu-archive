# 删除政策

## 第一条　总则

1. 本《删除政策》适用于本平台对内容下架、遮蔽、去标识化、限制访问等请求的受理与处理。
2. 本平台以《通用数据保护条例》（General Data Protection Regulation，以下简称“GDPR”）所确立的个人数据保护原则与数据主体权利框架为主要依据处理相关请求。若适用法律存在强制性规定或有效法律文书要求的，本平台将依强制性规定或法律文书处理。
3. 本平台的内容保存与展示以公共讨论记录的完整性与可追溯性为重要目标。除本政策明确允许的情形外，本平台不提供基于声誉管理、观点后悔、表达尴尬、评价不利等理由的删除服务。

## 第二条　术语定义

1. “内容”：指本平台展示、存储或处理的文本、链接、图片引用、元数据、索引、摘要、推荐结果及其结构化信息。
2. “个人数据”：指与已识别或可识别的自然人有关的任何信息。
3. “敏感个人数据”：指一旦泄露可能对自然人权益造成较高风险的信息，包括但不限于身份证件号码、精确住址、电话号码、账户凭证、支付信息、精确定位信息等。
4. “数据主体”：个人数据所对应的自然人本人。
5. “权利主张人”：提出请求的主体，包括数据主体本人或经数据主体明确授权的代理人。
6. “下架”：对外停止展示或限制展示。
7. “去标识化／遮蔽”：对个人数据字段进行遮盖、替换、散列化或其他处理，使其不再直接展示或难以识别对应自然人。

## 第三条　可受理的请求范围

本平台仅在符合下列条件之一时受理相关请求，并以最小必要原则处理：

1. 内容包含敏感个人数据，且该数据与数据主体可直接或可合理关联。
2. 内容包含未成年人敏感信息，可能对未成年人权益造成现实风险。
3. 内容包含账户安全信息（例如密码、验证码、令牌、私钥），可能导致账户或资产风险。
4. 存在明显的身份冒用或未经授权披露个人数据的情形，且权利主张人能够提供合理证据。
5. 收到具有约束力的法律文书或监管要求，且本平台能够核验其真实性与适用范围。

## 第四条　不予受理的请求类型

基于以下理由提出的请求，本平台原则上不予受理：

1. 仅因表达内容被认为“低质量”“不成熟”“不体面”“后悔发布”“影响形象”或类似原因。
2. 仅因观点争议、评价不利、被批评、被引用、被讨论等原因。
3. 要求删除公开讨论记录的全部内容或历史痕迹，且该请求不涉及敏感个人数据或不符合第三条规定。
4. 请求人无法证明其为数据主体本人或无法提供有效授权。
5. 请求明显超出合理技术可行性，或将对内容归档完整性、系统安全与服务稳定性造成不成比例影响。
6. 请求内容未被其他合法渠道处理（例如原发布平台未删除相关内容）。
7. 其他本平台认为不适合受理的情形。

## 第五条　处理方式与优先顺序

1. 在可受理情形下，本平台将优先采取对内容完整性影响最小的方式处理，包括但不限于去标识化、遮蔽、限制访问、断开敏感字段展示等。
2. 除非适用法律强制要求或本平台认为确有必要，本平台不承诺进行物理意义上的彻底删除。为安全审计、争议处理、反滥用与备份恢复之目的，本平台可能保留必要的日志与备份副本。
3. 本平台保留对处理方式、处理范围与处理时机作出最终决定的权利。

## 第六条　申请材料与身份核验

权利主张人应通过[《联系我们》](/contact)所列渠道提交请求，并至少提供：

1. 需处理内容的准确定位信息（链接、标识符、页面截图与标注）。
2. 所涉个人数据类型及其风险说明。
3. 身份核验材料：用于证明权利主张人身份与请求资格的合理信息。
4. 如为代理申请，应提供数据主体的明确授权证明。

本平台有权要求补充材料；材料不足的，视为未提出有效请求。

## 第七条　滥用与重复请求

1. 对明显恶意、重复、骚扰性或批量化的请求，本平台可直接拒绝或不再回复。
2. 本平台可保留必要证据，并在必要时采取技术或法律措施维护平台安全与他人合法权益。
